Proyecto: Formulario de Registro

Otro atributo 1 que se puede utilizar en los elementos <input> es el required. Cuando se agrega este atributo, el campo se vuelve obligatorio y debe ser llenado para que se pueda hacer el envío del formulario, de otro modo, el navegador mostrará un mensaje al usuario indicando que falta llenar el o los campos que tengan este atributo y se encuentren vacíos.

Como vimos en los pasos anteriores, ciertos valores del atributo ypet vienen con validación 2 del formulario pre-construida o nativa. Por ejemplo, type="email" 3 requiere que el valor sea una dirección de correo válida.

Con el atributo type="password" puedes utilizar el atributo pattern para definir una expresión regular 4 con la cual la contraseña debe coincidir para ser válida. Una expresión regular no son más que patrones que se utilizan para hacer coincidir combinaciones de caracteres en cadenas.

En la siguiente sección de nuestro formulario estaremos utilizando otro tipo de <input>, conocidos como botones radio, estos sirven para proporcionar multiples opciones a una pregunta, pero permiten que unicamente se pueda elegir una de estas opciones. Adicionalmente, estaremos utilizando el tipo checkbox 5, al igual que los tipo radio, pueden presentar multiples opciones, pero difieren en que permiten elegir más de una de estas opciones.
