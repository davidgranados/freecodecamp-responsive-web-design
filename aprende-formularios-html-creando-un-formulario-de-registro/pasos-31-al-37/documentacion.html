Proyecto: Formulario de Registro

Si se intenta enviar el formulario seleccionando cualquier opción del <select>, en realidad no va a proporcionar ningún tipo de valor útil al servidor. Para cambiar esto, cada elemento <option> debe tener el atributo value y el valor que se le asigne a este atributo, será el que en realidad reciba el servidor y se utilice como data para procesar del formulario. Sino, simplemente se enviará el texto que contiene el elemento <option> seleccionado.

También podemos recibir información por parte del usuario a través del elemento <textarea> 1. Estos actúan como los elementos <input> de tipo text, pero con la diferencia o beneficio adicional de que son capaces de recibir múltiples líneas de texto y además se les puede dar un valor inicial a sus dimensiones, las cuales vienen representadas por filas y columnas y se modifican a través de sus atributos rows y cols, respectivamente.

Además de los elementos <label>, donde damos al usuario información sobre el campo que está llenando, podemos utilizar el atributo placeholder. Este atributo acepta un valor de texto, el cual es mostrado dentro del campo o elemento <input> hasta que el usuario comienza a escribir dentro de dicho elemento.

El atributo name se utiliza para identificar cada elemento al momento de enviar el formulario, es decir, cuando se recibe el formulario luego de ser enviado, es procesado utilizando el valor de su atributo name. Es por esta razón que es una buena práctica siempre asignar un valor a cada elemento del formulario que se desea procesar.

Comenzaremos a dar estilos a los elementos de nuestro formulario, comenzando por los que están dentro del <body>. Modificaremos la familia de fuentes y su tamaño modificando sus propiedades font-family 2 y font-size 3.
